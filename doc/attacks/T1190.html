T1190 - Exploit Public-Facing Application

Описание атаки

Эксплуатация уязвимостей в публично доступных приложениях, таких как веб-серверы, сервисы баз данных, приложения для совместной работы. Атакующие ищут и используют известные уязвимости (CVEs) в публичном ПО для получения первоначального доступа к сети.

Принцип работы

  • Сканирование на наличие уязвимых приложений
  • Использование известных эксплойтов (метасплойт и т.п.)
  • Эксплуатация zero-day уязвимостей
  • SQL инъекции, XSS, RCE в веб-приложениях
  • Уязвимости в CMS (WordPress, Joomla, etc.)
  • Эксплуатация уязвимостей в сетевых сервисах

Защита с помощью Security Manager

smnet - Network Monitoring

  • Мониторинг внешних подключений: smnet connection
  • Обнаружение подозрительного трафика: smnet stats
  • Сканирование открытых портов: smnet scan
  • Регулярный аудит сетевой активности: помогает обнаруживать эксплойты

smlog - Log Analysis

  • Анализ веб-сервер логов: smlog search "error|exploit" /var/log/apache2/access.log
  • Мониторинг системных ошибок: smlog monitor
  • Отслеживание подозрительных запросов: smlog search "union|select|script" /var/log/nginx/access.log

smssh - SSH Security

  • Защита от secondary access: smssh apply /etc/ssh/sshd_config
  • Мониторинг SSH после компрометации: smssh monitor
  • Генерация ключей для безопасного доступа: smssh gen-key

smpass - Password Management

  • Управление учетными данными приложений: smpass add-password
  • Безопасное хранение credential: smpass с AES256 шифрованием

Рекомендации

  • Регулярное обновление всех публичных приложений
  • Использование WAF (Web Application Firewall)
  • Сканирование на уязвимости (Nessus, OpenVAS)
  • Минимизация attack surface
  • Использование контейнеризации для изоляции
  • Регулярные пентесты и аудиты безопасности
  • Мониторинг на аномальную активность
  • Использование principle of least privilege
